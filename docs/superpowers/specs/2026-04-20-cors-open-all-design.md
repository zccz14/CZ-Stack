# API 全局开放 CORS 设计说明

## 背景 / 问题

当前 `modules/api/src/app.ts` 使用 Hono 创建 API 应用，并注册了 `/health` 与 `/openapi.json` 两类公开接口，但尚未配置任何 CORS 中间件。

这会带来两个直接问题：

1. 浏览器环境访问这些接口时，跨域请求与 `OPTIONS` 预检请求缺少统一响应，前端联调体验不稳定。
2. 模板仓库未明确表达“CORS 不是后端访问控制机制”，容易让后续实现误把浏览器跨域限制当成服务端安全边界。

本次设计目标是在 API 层统一开放 CORS，对所有来源放行，并把该选择明确为模板默认行为。

## 目标与非目标

### 目标

1. 为 `@cz-stack/api` 的所有路由启用全局 CORS 中间件。
2. 对所有来源返回允许跨域访问的响应头，`origin` 固定为 `"*"`。
3. 确保普通请求与浏览器 `OPTIONS` 预检请求都能得到一致、可预期的 CORS 行为。
4. 在仓库文档或实现附近明确说明：CORS 只解决浏览器跨域兼容性，不承担后端访问控制职责。

### 非目标

1. 不引入按环境、域名、租户或路径维度的 origin allowlist。
2. 不把 CORS 配置扩展为鉴权、ACL、内网隔离或其他访问控制方案。
3. 不新增用于开关 CORS 的运行时配置项。
4. 不修改现有路由语义、响应体结构或 OpenAPI 契约内容。

## 方案设计

在 `modules/api/src/app.ts` 的应用级入口注册一次 Hono 官方 CORS 中间件，并作用于全量路由。

设计约束如下：

1. 中间件应在路由注册前生效，确保 `/health`、`/openapi.json` 以及未来新增接口都自动继承同一策略。
2. `origin` 固定为 `"*"`，不引入白名单、动态回调或条件分支。
3. 保持配置最小化，只覆盖“允许所有来源访问”的明确需求；不借机引入模板暂时不需要的复杂 header 策略。
4. 在代码注释或配套文档中明确声明：CORS 是浏览器兼容层，不是后端授权边界；服务端若需要限制访问，必须依赖独立的鉴权或网络层机制。

该方案的核心收益是把跨域行为收敛到应用入口，避免逐路由重复配置，也避免未来新增接口时遗漏预检支持。

## 路由行为与预检请求

引入全局中间件后，路由行为应统一如下：

1. 对现有 `GET /health` 与 `GET /openapi.json`，正常响应中包含允许跨域访问所需的 CORS 响应头。
2. 对任意已注册路径的浏览器跨域请求，若触发 `OPTIONS` 预检，请求应由同一套全局 CORS 配置处理，而不是要求每个业务路由单独声明 `OPTIONS` 处理器。
3. 对未来新增接口，默认继承相同 CORS 行为，无需额外逐条配置。
4. CORS 放行仅表示浏览器可发起跨域调用，不代表接口对匿名用户“自动安全”或“自动受限”。

模板层面的预期是：浏览器访问 API 时，不应因为缺失预检支持或缺失 `Access-Control-Allow-Origin` 而被拦截；真正的访问权限仍由后端业务逻辑、鉴权和部署网络策略决定。

## 测试与验证策略

本次以最小必要验证覆盖全局行为：

1. 为 API 应用入口补充测试，确认普通请求响应包含预期的 CORS 头。
2. 补充 `OPTIONS` 预检用例，确认已有公开路径可返回成功的预检响应，而不是落入 404 或未声明方法。
3. 测试至少覆盖一个现有公开接口，用于证明全局中间件已生效；首选 `/health`，如现有测试布局更适合，也可覆盖 `/openapi.json`。
4. 保持验证聚焦在“全局开放 CORS”本身，不把测试扩展成鉴权、代理或浏览器端集成测试。

## 风险与兼容性

当前方案风险较低，但需明确以下边界：

1. `origin: "*"` 适用于当前模板“默认公开 API”定位；如果后续出现基于 Cookie 的凭证场景，应重新评估 CORS 与凭证策略的组合约束，而不是在本次设计内提前泛化。
2. 全局开放 CORS 会让任何浏览器来源都能调用公开接口，因此团队不能把“前端页面不在白名单里”视为访问限制手段。
3. 该变更属于兼容性增强，一般不会影响非浏览器客户端；命令行、服务间调用和现有路由语义保持不变。

## 受影响文件

1. `modules/api/src/app.ts`：注册全局 CORS 中间件。
2. `modules/api` 下的应用入口测试文件：补充普通请求与 `OPTIONS` 预检覆盖。
3. `docs/api/README.md`：补充模板层面的约束说明，明确 CORS 只用于浏览器跨域兼容，不用于限制后端访问。
